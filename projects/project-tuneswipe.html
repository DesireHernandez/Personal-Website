TuneSwipe

Fall 2024 - Spring 2025 Group Project 10 Months
Mobile App Recommendation System React Native Machine Learning

TuneSwipe is a mobile application designed to revolutionize music discovery through an intuitive swipe-based interface. By leveraging machine learning algorithms, TuneSwipe offers personalized music recommendations, allowing users to effortlessly explore new tracks and artists tailored to their unique tastes.

View Code

Project Overview

In today’s streaming era, music recommendation algorithms often trap users in echo chambers, repeatedly suggesting the same styles, artists, or overplayed tracks. TuneSwipe was built to challenge that pattern by putting discovery control back in the user’s hands through an engaging, swipe-based experience that makes finding new music feel intentional rather than overwhelming.

The application leverages behavioral analysis by tracking how users interact with songs, artists, and genres throughout their discovery journey. By combining direct feedback like swipes and likes with Spotify listening history and audio features, TuneSwipe generates personalized playlists that balance familiarity with exploration, helping users uncover music that actually evolves with their taste.

Key Features

Intelligent Swipe Interface

Swipe right to like or left to dislike songs, artists, or genres using a card-based interface. Each swipe captures direct feedback that immediately influences future recommendations and playlist generation.

Behavioral Analysis

Tracks user interactions such as swipes, likes, dislikes, and listening behavior to build a dynamic preference profile that evolves over time.

Personalized Playlists

Automatically generates playlists tailored to each user’s taste by combining swipe data with Spotify listening history, then saves and syncs those playlists directly to the user’s Spotify account.

Discovery Zones

Explore music through dedicated modes for songs, artists, or genres, giving users control over how they discover and curate new music.

Real-Time Listening Feedback

Users can like or dislike songs as they play, allowing TuneSwipe to capture immediate feedback and continuously refine recommendations without interrupting the listening experience.

Seamless Spotify Integration

Authenticates users through Spotify OAuth, syncs playlists directly to their Spotify library, and uses real listening history to power recommendations.

Preference Management & Customization

Users can manually manage liked and disliked songs or artists, update profile details, and customize their experience to further refine recommendations.

Architecture

TuneSwipe follows a client-server architecture where a React Native mobile application communicates with a Supabase backend and integrates directly with Spotify’s Web API and SDK. The system is designed to capture user interactions in real time and translate them into personalized recommendation outputs.

  • Moblie Client: Built with React Native and TypeScript for cross-platform compatibility. The client handles the swipe interface, playback controls, and user interactions while securely communicating with backend services and Spotify’s APIs.
  • Backend & Database: Supabase provides authentication, database management (PostgreSQL), and API endpoints. User preferences, swipe history, and interaction data are stored and structured to support dynamic playlist generation.
  • Machine Learning & Data Analysis: Python-based experimentation using scikit-learn (PCA and K-Means) was conducted to analyze Spotify audio features and explore clustering-based recommendation strategies.
  • APIs: Spotify Web API and Spotify SDK enable OAuth authentication, music metadata retrieval, playback control, and direct playlist syncing to user accounts.

Recommendation Algorithm

TuneSwipe uses a hybrid recommendation approach centered on explicit user feedback and content-based filtering to generate personalized playlists.

  • Content-Based Filtering: Spotify audio features such as tempo, energy, danceability, valence, and acousticness are analyzed to identify songs with similar characteristics. Dimensionality reduction (PCA) and clustering techniques (K-Means) were explored using scikit-learn to group tracks by similarity and better understand feature relationships.
  • Explicit Preference Modeling: User swipes (like / dislike) directly influence future recommendations. Liked songs and artists increase the likelihood of similar tracks appearing in generated playlists, while disliked songs and artists are strictly excluded from the current playlist creation.
  • Manual Preference Controls: In addition to swiping, users can manually add songs or artists to their liked or disliked lists. This ensures the system respects strong user preferences and prevents unwanted content from resurfacing.
  • Hybrid Stategy: The final playlist generation logic combines audio feature similarity with explicit preference filtering, balancing discovery with user-defined constraints to maintain both personalization and variety.

Challenges and Solutions

Challenge: Cold Start Personalization

New users with limited interaction data required an effective way to generate personalized recommendations without defaulting to generic popular tracks.

Solution: Incorporated Spotify listening history and initial user interactions to seed recommendations for new users. Early swipe input was prioritized to quickly buuld a preference profile and influence playlist generation from the first session.

Challenge: Asynchronous Data Flow & Supabase Integration

The application relied heavily on asynchronous operations, including API calls, database updates, and activity log tracking. Managing nested user activity logs and ensuring consistent database updates required careful sequencing.

Solution: Used async/await to control execution order and centralized loading state management to reduce UI inconsistencies. Updated nested activityLog data by retrieving the existing record, modifying the appropriate session data, and safely writing the full updated structure back to Supabase. Extensive logging helped trace data flow and quickly identify integration issues.

Challenge: Responsive UI & User Experience

Swipe cards and interface components initially did not scale properly across different device sizes, impacting usability.

Solution: Used the React Native Dimensions API to dynamically size components and tested across multiple screen sizes. Added loading indicators (ActivityIndicator) and form validation logic for playlist creation to improve responsiveness and overall user experience.

Results & Impact

TuneSwipe demonstates a full-stack mobile application that transforms user input into real, usable music experiences by generating personalized playlists that sync directly to Spotify. The system supports an end-to-end workflow from user interaction to playlist creation, showcasing the intergration of mobile development, backend services, machine learning, and third-party APIs. The application generates playlists of up to 20 songs based on user swipe behavior and explicit preferences, allowing users to quickly build collections that reflect their evolving music tastes. By combining Spotify audio features with direct user feedback, TuneSwipe provides a more interactive and controlled alternative to traditional recommendation systems. User testing during development highlighted the effectiveness of the swipe-based interface in making music discovery more engaging and intuitive. The ability to explicitly like or dislike songs and artists ensured that recommendations remained relevant while filtering out unwanted content.

Lessons Learned

This project reinforced several important lessons about building user-facing applications:

  • Designing for Real-Time Interaction: Building a swipe-based interface taught the importance of responsiveness and immediate feedback. Even small delays in updating UI state or syncing with external APIs significantly impacted the user experience, reinforcing that fluid interactions are critical in mobile applications.
  • Managing Asynchronous Systems: Working with multiple asynchronous operations, such as Spotify API calls, Supabase updates, and playback synchronization, highlighted the complexity of managing data flow. This project strengthed my understanding of async/await patterns, error handling, and maintaining consistent application state.
  • Structuring and Updating Complex Data: Managing nested user data, such as session-based activity logs, required careful handling to avoid overwriting or corrupting existing data. This improved my understanding of working with structure data in databases and safely performing updates in a production-like environment.
  • User Experience Goes Beyond UI: Features like loading indicators, responsive layouts, and input validation proved essential for usability. This reinforced that a polished user experience depends just as much on system feedback and perforance as it does on visual design.
