TuneSwipe is a mobile application designed to revolutionize music discovery through an intuitive swipe-based interface. By leveraging machine learning algorithms, TuneSwipe offers personalized music recommendations, allowing users to effortlessly explore new tracks and artists tailored to their unique tastes.

Project Overview

In today's streaming era, music recommendation algorithms often trap users in echo chambers, repeatedly suggesting similar tracks without exposing them to genuinely new sounds. TuneSwipe addresses this problem by putting discovery control back in the user's hands through an engaging, gamified experience.

The application leverages behavioral analysis, tracking not just what users like, but how quickly they swipe, how long they listen, and patterns in their discovery journey. This nuanced data creates highly personalized playlists that balance familiarity with exploration.

Architecture

The applicatin follows a microservies architecture with a React Native front-end communicating with a Supabase backend. User interactions are captured in real-time, with behavioral data processed using Python-based machine learning models to generate dynamic playlists.

  • Moblie Client: Built with React Native for cross-platform compatibility, featuring smooth animations and responsive gesture controls.
  • Backend: Supabase handles user authentication, data storage, and real-time updates.
  • Machine Learning: Python scripts utilizing Scikit-learn analyze user behavior to refine recommendation algorithms.
  • APIs: Integration with the Spotify API and SDK allows seamless music playback and access to a vast music library.

Recommendation Algorithm

The recommendation engine combines multiple data sources and techniques:

  • Content-Based Filtering: Analyzes audio features (tempo, key, energy, danceability) and metadata of tracks to suggest similar songs.
  • Collaborative Filtering: Leverages community listening patterns to recommend tracks favored by users with similar tastes.
  • Behavioral Analysis: Tracks swipe speed, listening duration, and skip rates to fine-tune recommendations based on user engagement.
  • Hybrid Approach: Combines content-based and collaborative filtering with behavioral insights for a holistic recommendation strategy.

Lessons Learned

This project reinforced several important lessons about building user-facing applications:

  • User Experience First: The simplest interaction model often wins. The swipe interface needed to be instantaneous and satisfying.
  • Data Privacy Matters: Beign transparent about what data we collect and why built trust with users.
  • Iterate with Feedback: Early prototypes had too many features. Simplifying based on user testing led to a more focused, polished product.
  • Performace is a Feature: No amount of clever algorithms matter if the app feels sluggish. Optimiztion was crucial.
